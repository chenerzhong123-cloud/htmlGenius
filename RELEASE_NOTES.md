# 更新日志

> 本文件记录 htmlGenius 所有已发布版本对**用户**可见的变化。
> README 仅保留最近三个版本，完整历史在此。

---

## v0.9.2（2026-07-25）

- **本地连接组件更可信**：`@htmlgenius/bridge` 现可在 npm 页面一键溯到公开 GitHub 源码，附带包级说明与 MIT 协议；发布改用 CI 带 **provenance（来源证明）**，安装时的「陌生新包」风险提示显著降低。
- **连接中心默认收起**：Connection Center 现在默认折叠（无论是否已连接），标题仍显示状态，点一下即可展开，不再占用契约页空间。
- **置灰有说明**：在远程网页上「发送给 Agent」按钮置灰时，会提示「直接发送仅支持本地 HTML 文件，可改用复制 Prompt」，不再让人摸不着头脑。
- **编辑页精简**：移除编辑帮助栏位、artifact 信息行（文档 ID / 基线哈希）与「重新读取文件」按钮；暂时隐藏较难懂的「切换高级模式」入口（元素级编辑能力保留）。
- **文案优化**：选择修改范围页提示改为「不指导修改文章的评论，扔掉效果更好」。
- **安装门槛放宽**：本地连接组件现支持 Node 20.x 与 22+（此前要求 20.19+/22.12+）；仅 GitHub Copilot 仍需 20.19+/22.12+，不满足时 Copilot 单独显示「不兼容」，不影响 Codex / Claude Code。

## v0.9.1（2026-07-23）

- **Agent 支持更可靠**：新增 Agent 现在必须通过统一的「能力认证」——探测、生成候选、生成计划、安全边界逐项验证通过后才会出现在你的发送菜单里，减少「显示支持却用不了」的情况。
- **连接状态更清楚**：Connection Center 逐个 Agent 给出真实状态（已连接 / 未检测到 / 需登录 / 版本不兼容）与对应做法；「复制诊断」只给脱敏信息，不含路径与隐私。

## v0.9（2026-07-23）

- **本地连接，面向所有人**：不再需要进源码仓库跑脚本。未连接时契约页出现 **Connection Center**：点「**让 Agent 帮我连接**」复制一段严格限定的安装说明粘贴给你正在用的 Agent，或「复制 Terminal 命令」自己执行——一条命令即可装好本地连接组件。
- **连接状态一目了然**：区分「未安装 / 需修复 / 组件就绪但 Agent 未登录 / 已连接 N 个 Agent / 系统不支持」，逐项给真实状态与官方登录指引；「检查连接」只读重探。
- **安全修复**：仅在判定可自修时出现「安全修复」（二次确认），明确只重写 Chrome 注册文件，不安装 Agent、不改你的项目文件。
- **没连上也能用**：任何状态下都可「复制 Prompt」手动交给任意 AI。
- 本地连接组件以 `@htmlgenius/bridge` 发布到 npm；仅 macOS + Chrome + Node 20.19+/22.12+。

## v0.8.2（2026-07-23）

- **新增 GitHub Copilot**：发送菜单第三个 Agent。优先用你本机的 `copilot` 命令行；没装时自动用 Copilot SDK 自带运行时，菜单实时显示所用方式。
- **受控安全边界**：Copilot 运行在受限模式——只开放文件读写并逐个校验路径（候选只允许写候选文件、计划只允许写计划文件），shell / 网络 / 子任务全禁；每次发送都是一个新会话，**绝不读取或续发你在 Copilot / VS Code 里的已有会话**。
- **安装更宽松**：装本地连接组件不再要求本机有 Claude CLI——只用 Copilot 或 Codex 也能装。

## v0.8.1（2026-07-22）

- **Codex 接入（推荐优先）**：除 Claude Code 外，可把修改契约交给本机 Codex（ChatGPT.app 内置）。Codex 是常驻热服务，握手快、逐字流式输出，整页重做通常几分钟完成；Claude Code 走命令行冷启动，等待更久属正常。两者都只写候选、永不覆盖原文件，只用本机登录态、不存任何凭证。
- **修改范围三档直达**：「基于评论修改文档」默认全选评论，直达「选择修改范围」——精确修补 / 局部优化 / 全文重做三档；全文重做不再强制填 brief，评论即输入。
- **Mint 深色主题**：整个侧边栏与页面级刷新弹窗统一为 Mint 配色。
- **候选版本号**：每次生成写入文件名（`原名V1.1.html`、`V1.2.html`…按文档累计），状态栏显示「候选 V1.N 已生成」。
- **生成过程可感知 + 可中止**：发送后按钮变「终止任务」（可随时中止）；状态栏自动展开显示 Agent 逐字输出与文件 / 命令 / 思考事件 + 计时器，完成后自动收起。
- **完成即开 + 通知**：自动新开页签打开候选 + 系统通知；回原 tab 发送按钮自动恢复，便于改评论后再发下一版。
- **多 tab 独立**：每个浏览器 tab 一份契约 / 运行态，切 tab 自动跟随。

## Night Pack A · 候选闭环（2026-07-20）

- **Claude 产出候选 HTML**：发送给 Claude Code 后，本机 Agent 在受控工作区写 `candidate.html`（**只放行 Read/Glob/Grep/Write，仍禁 Bash/Edit/网络/MCP，无 shell**），host 校验后把候选复制为源文件同级的 `<原名>--htmlgenius-<runId>.candidate.html`；**源文件全程不被覆盖**。
- **source 快照 + 双哈希**：运行前把源复制为只读快照（0400）并校验哈希；运行后再读源哈希，运行期被外部改动即 `SOURCE_MUTATED_DURING_CANDIDATE`、候选不被采用、不显示成功。
- **候选形态校验**：候选必须存在、非 symlink、非空、不超上限、且是真正的 HTML（`<!doctype html>`/`<html>`），否则拒绝（阻止模型写出 Markdown / 纯文本）。
- **打开候选 + 评论重锚**：候选经 v0.6.2 受控 `new_artifact` 路径打开（background 逐字段比对 run/task/source 哈希后才放行），新页评论自动重定位。
- **最小只读成功态**：侧边栏显示「候选版本已生成 / 原文件未被修改 / 已重锚 X/Y 条，Z 条待检查 / provider·时间·源→候选文件名·源哈希已校验·候选 hash 前缀」，并提供「打开候选 / 返回原文件」。**这不是「接受修改」**——无 promote / 覆盖 / 自动接受按钮或路径。
- **持久证据**：刷新侧边栏后，最近一次候选的证据仍可显示；run 记录只存元数据（候选 URI / 候选 hash / manifest 路径），**不存 prompt / 评论 / 候选 HTML / Claude 回复**。
- 真实 Claude 运行的 smoke 测试标记为「待用户手动验证」（夜间不消耗本地额度）；fake CLI 自动化测试覆盖写候选 / 写 Markdown 拒绝 / source 突变拒绝 / argv 注入安全。

## v0.7.2（2026-07-20）

- **评论 → 编辑任务的选择流**：打开评论页默认仍是完整评论收件箱（先讨论）；底部新增次级入口「**整理评论，创建编辑任务**」。点击后先在侧边栏内进入「**挑选这次要处理的评论**」步骤：默认全选所有顶层且未失效的评论，用户可手动取消本次不交给 Agent 的；回复随被选 root 完整带出，不单独勾选；超过 20 条给出分批提示。**没有任何模型或规则替用户判断一条评论是不是修改建议。**
- **状态可回退**：挑选 → 契约表单之间可往返，返回时保留本轮勾选与已填内容；关闭或 Esc 则清空临时选择与表单，再次进入恢复默认全选。未选/已失效评论绝不进入 Prompt 或 JSON。
- **模式文案更易懂**：四种修改目标改为「只改圈选处 / 优化这一区块 / 先给我改法 / 重做整页」，仍是互斥的四选一（不偷改 schema）；字段改为「你想让页面变成什么样？」「有绝对不能动的内容吗？（可选）」，长解释收进可键盘聚焦的 `?` tooltip。
- **布局**：评论列表与选择列表超出首屏只纵向滚动，底部操作按钮常驻第一屏；窄 Side Panel 下长文本换行、tooltip/菜单向内展开，**不出现横向滚动条**。
- 三语（中/英/日）i18n 完整；既有评论创建/回复/编辑/删除/定位、Change Contract、Claude handoff 全部不回归。

## v0.7.1（2026-07-20）

- **一键交接 Claude Code**：修改任务 Composer 中「精准修补 / 局部优化 / 重新生成」三种模式出现「**发送给 Claude Code**」——本机已登录的 Claude Code CLI（先 `claude auth login`）一键收到完整修改契约，不用切换应用、不用复制粘贴。「结构重组」仍只复制规划任务。
- **会话续发**：可选「继续 HTML Genius 上次创建的 Claude task」；只续发插件自己创建的 Claude 任务，绝不读取、枚举或注入你其他的 Claude 会话。
- **本版为交接验收版**：Claude 只确认收到任务，**不修改任何文件**（源文件在交接前后做 SHA-256 双校验，有变化即失败并报因）。真正的受控编辑执行见后续版本。
- **本地传输证据**：完整契约落盘 `<源文件目录>/.htmlgenius-bridge/claude/<文档ID>/task-*.{json,md}`（0600 权限），可本地审计。
- **安装**：`cd bridge && node install-macos.mjs --extension-id <扩展ID>`，详见 `docs/LOCAL_BRIDGE.md`。仅 macOS + Node 20+；host 名改为 provider-neutral 的 `com.htmlgenius.local_bridge`（后续 Codex adapter 复用）。

## v0.8（2026-07-20）

- **高亮不再「盖字」**：页面工具栏与侧边栏的调色板统一为同一份取值，并移除会让浅色文字隐形的纯白高亮；对失效选区施色不再静默无反应，而是明确提示「请先在页面选中文字」。
- **重做真正可用**：此前「打字后马上点撤销」「工具栏改色/改字号」会导致改动丢失、redo 回不来；现在撤销/重做完全对称，工具栏的所有编辑（颜色 / 字号 / 标题 / 对齐 / B/I/U/S / 清除格式）都记入撤销历史。
- **侧边栏工具补齐**：新增评论 / B/I/U/S / 清除格式 / 字号 / 标题 / 对齐，与页面工具栏走**同一份修改逻辑**（唯一入口），B/I/U/S 点亮状态两边实时同步。
- **图标统一**：侧边栏文字色（A 带红底条）/ 高亮色（荧光笔黄笔头）图标与页面工具栏完全一致。
- **选色弹窗整齐**：改为整面板宽浮层，8 列 × 2 行色格无空位，不再超出面板右边界。
- **「生成修改任务」常驻第一屏**：评论再多，该按钮固定在底部；评论区独立滚动（底部操作区为可扩展容器）。
- **编辑文字立即出光标**：高级模式点「编辑文字」后立即出现闪烁光标（不用再点一下控件），选框同时闪一圈作反馈。
- **修复「刷新」按钮**：激活确认弹窗点「刷新」偶发无反应的问题已修复（即便页面拦截了刷新也会兜底进入编辑）。
- **评论文案统一**：「批注 / 评论」不再混用，全部统一为「评论」（中 / 英 / 日）。

## v0.6（2026-07-16）

- 多项编辑体验修复（选区高亮、工具栏与编辑态联动等）。

## v0.5.2（2026-07-15）

- **撤销 / 重做 / 还原**：编辑历史改为线性模型，撤销（Cmd/Ctrl+Z 或侧栏按钮）与重做真正可用；新增「还原到本次编辑初始」一键回基线。本地页与远程页（临时编辑）均生效。
- **侧边栏编辑工具栏**：编辑态新增撤销 / 重做 / 还原初始 / 另存为 HTML 四个会话动作；「另存为」把当前正文下载为 .html（本地/远程均可，无法覆盖原文件）。
- **原生色环取色**：侧边栏文字色 / 高亮色改用系统原生取色器（自由选色）；浮窗色板保持不变。
- **点击高亮聚焦评论**：在页面点已有批注的高亮，侧边栏自动切到批注 Tab、滚到该卡片并聚焦回复输入。
- **在线人数移至身份入口**：评论区不再显示「在线：姓名」，改在账号浮层显示「本页在线 N 人」；登出即清。
- **编辑模式 alert 气泡**：编辑视图顶部用标准 alert 组件（本地=绿「可保存」/ 远程=黄「临时」），浅色主题用标准暖黄/浅绿。
- **修复撤销失效**：此前撤销监听仅在 file:// 本地页注册、且首次撤销空操作；现已全局可用。

## v0.5.1（2026-07-14）

- **深色 / 浅色主题切换**：侧边栏右上角一键切换；深色 Nebula 风格，浅色参照 Airtable 设计规范（白底 + 近黑主按钮）。浮动工具栏与确认弹窗跟随切换；无偏好时跟随系统 `prefers-color-scheme`。
- **中 / 英 / 日 多语言**：跟随浏览器语言，可在头像旁切换并本地记住。
- **评论可编辑**：作者可编辑自己已保存的评论。团队模式经后端 `PATCH /api/annotations/:id`（作者校验，跨团队/非作者 403），编辑经 SSE 实时同步给队友。
- **收起侧边栏即停止**：关掉侧边栏后，浮动工具栏 / 高亮 / 编辑态立即消失（端口长连接断开触发），页面恢复正常浏览。
- **失效评论一键清理**：网页内容变动后找不到原文位置的旧评论，归档到侧边栏底部，可一键全部删除。
- **修复本地 HTML 版本陈旧**：若本地 HTML 被外部（如另一个 agent）改过，打开插件时展示**新版本**而非带旧评论的旧版本（基于正文哈希判定）。
- 编辑工具栏：加粗 / 斜体 / 下划线 / 删除线支持切换高亮与共存；修复字号、对齐不生效；文字颜色、荧光笔换标准 SVG 图标。
- 后端 CORS 放开 `PATCH` 方法。

## v0.1（Beta，2026-06-25）· 首个版本

- **划词批注**：选中文字 → 写下评论。
- 高亮**不破坏原网页**。
- **批注侧边栏**集中展示所有批注。
- AI 改完出新版本后，批注**自动跟到新位置**；原文被删则归档保留。
- **一键回灌**：把所有批注打包成给 AI 的指令并复制到剪贴板。
