htmlGenius

支持的 Agent 与使用方法

htmlGenius 让你本机已登录的 AI Agent 按评论改 HTML 文档。它只生成候选版本，从不覆盖你的原文件；评论与文档内容都只在本机处理，不会上传到任何第三方服务器。

支持的 Agent

Codex 推荐优先

通过 Codex Mac App（ChatGPT.app 内置的 codex app-server）执行

  • 更快：连的是已经在运行的常驻进程，handshake 几乎零开销；token 逐字流式输出，你能实时看到进度。
  • 更稳：单轮流式 turn 直接写出候选文件，往返少，整页重做也通常几分钟内完成。

如何启用：

  • 安装 Codex Mac App（ChatGPT.app，bundle id com.openai.codex）并登录。
  • htmlGenius 会自动发现并校验 App 签名（TeamID），无需任何额外配置。
  • 在发送按钮右侧的 ⌄ 里选择 Codex 即可。

Claude Code 同样支持

通过官方 claude 命令行（CLI）执行

  • 复用你本机 Claude Code 的登录态，不读取或保存任何 API Key / Token / Cookie。
  • 在只读工具白名单 + 沙箱内工作，只能写候选文件，禁止 Bash / 联网 / MCP。

如何启用：

# 安装 claude CLI 后登录（只需一次）
claude auth login

登录后在 ⌄ 菜单里选择 Claude Code 即可。

为什么 Claude Code 会更慢（这是正常的）
Claude Code 每次任务都会新起一个 claude 子进程：冷启动要加载配置、初始化、校验登录，首 token 前有几秒到十几秒开销，且每发一次都付一次冷启动税；它还可能走多轮工具循环（读文件 → 规划 → 分步写）。相比之下 Codex 是热服务，所以 Claude Code 的等待时间会明显更久，并非卡死。如果赶时间，优先用 Codex。

安全与隐私边界

  • 永不覆盖原文件：Agent 只在受控工作区写出候选版本（原名V1.1.html 这样的独立文件）。
  • 不存凭证：只复用本机已有的 Agent 登录态，绝不收集或上传 API Key / Token / Cookie。
  • 不接管你的会话：每次都新建独立任务，不读取 / 列出 / 注入你外部正在跑的 Codex 或 Claude 会话。
  • 沙箱：Codex 用 workspace-write + 禁网；Claude 仅 Read/Glob/Grep/Write，禁 Bash / Edit / 联网 / MCP。
  • 状态栏可见：发送后可展开看到实时进度；任何时刻可点「终止任务」中止。

htmlGenius · 候选版本以 V1.1 / V1.2 命名，按文档累计，便于你回溯每一版改动。
