## 一、自动化后端验收（Codex 直接跑，已验证全绿）

脚本：`scripts/accept_v05.py`。覆盖 v0.5 所有后端保证。

```bash
HG_BASE=https://www.deuce.monster/hg uv run python scripts/accept_v05.py
# 退出码 0=全过,1=有失败
```

用例（脚本内顺序执行）：

| # | 用例 | 通过条件 |
|---|---|---|
| 1 | `GET /health` | `{"status":"ok"}` |
| 2 | `GET /auth/me`（带 token） | 返回 session 的 `open_id`+`name` |
| 3 | `GET /auth/me`（无 token） | 401 |
| 4 | `POST /api/annotations`：author 字段伪造 | 返回的 `author` 来自 **session**，请求体里的被忽略 |
| 5 | 批注 `team_id` | = session 的 team（`default`） |
| 6 | **非作者 `DELETE`** | **403**（v0.5 硬身份核心） |
| 7 | 非作者删失败后 | 批注仍在 |
| 8 | **作者 `DELETE`** | **200** |
| 9 | 无 session 建批注 | 401 |
| 10 | `POST /auth/logout` 后原 token | `/auth/me` → 401 |
| 11 | **SSE 实时推送**：A 订阅 doc，B 在同 doc 建批注 | A 收到 `annotation:created` |

> 脚本里 SSE 用 `curl -N` 子进程读流（httpx/http.client 在小 chunk SSE 上有缓冲问题，curl 稳）。运行环境需 `curl`（macOS/Linux 均自带）。

## 二、快速 curl 验证（单条命令，硬身份核心）

整段粘进终端，验证「B 无法删 A 的批注（403），A 自己能删（200）」：

```bash
BASE=https://www.deuce.monster/hg
A=$(curl -s -XPOST $BASE/auth/dev-login -H 'Content-Type: application/json' \
    -d '{"open_id":"ou_alice","name":"Alice","team":"default"}' | python3 -c 'import sys,json;print(json.load(sys.stdin)["token"])')
B=$(curl -s -XPOST $BASE/auth/dev-login -H 'Content-Type: application/json' \
    -d '{"open_id":"ou_bob","name":"Bob","team":"default"}' | python3 -c 'import sys,json;print(json.load(sys.stdin)["token"])')
ANN=$(curl -s -XPOST $BASE/api/annotations -H "Authorization: Bearer $A" -H 'Content-Type: application/json' \
    -d '{"document_id":"doc_smoke","selector":{"type":"TextQuoteSelector","exact":"hello"},"quote":"hello"}')
echo "$ANN"
ID=$(echo "$ANN" | python3 -c 'import sys,json;print(json.load(sys.stdin)["id"])')
curl -s -o /dev/null -w "Bob 删 → HTTP %{http_code} (期望 403)\n" -XDELETE $BASE/api/annotations/$ID -H "Authorization: Bearer $B"
curl -s -o /dev/null -w "Alice 删 → HTTP %{http_code} (期望 200)\n" -XDELETE $BASE/api/annotations/$ID -H "Authorization: Bearer $A"
```

## 三、协同 UX 验收（载体 = 扩展，**不是** viewer）

> ⚠️ 重要更正：**viewer web app 不是协同 UX 的等价验证载体**。viewer 是 v0.2 遗留的「单用户批注/回灌」页面，**没有**实时 SSE 订阅、回复 UI、作者感知删除——这三项是**扩展独有**（`extension/sync.js` + `sidepanel.js`）。早期文档把 viewer 当协同等价物是过度声明，已纠正。

**viewer 现在能做什么**（base-path 已修，可在 `/hg` 子路径下加载）：
- 单用户：加载样本文档 → 划词批注 → 列表 → 一键回灌。配合 dev-login 把 session token 粘进 viewer 的 token 框，可在部署服务器做**单用户只读冒烟**。
- **不覆盖**：实时同步、多级回复、presence、作者感知删除（这些 viewer 从未实现，非 v0.5 回归）。

**真正的协同 UX 验收走扩展**（目前只能人工：真 Chrome + 飞书；或扩展 E2E）：

| # | 用例 | 验收方式 | 通过条件 |
|---|---|---|---|
| U1 | 实时同步 | 两浏览器/账号(同 URL),A 批注 | B 不刷新,秒级出现高亮+卡片 |
| U2 | 多级回复 | B 在 A 卡片 hover→回复 | A 出现子卡;DFS 层级正确 |
| U3 | presence | A 登录进页面 | B 侧栏显示「在线:A」 |
| U4 | 硬删除 UI | A 建批注 | B 卡片**无删除按钮**(`ann.author.id !== me`,扩展侧栏已实现) |
| U5 | 作者删除级联 | A 删带回复的父批注 | 整棵子树从 B 消失 |
| U6 | 归档(stale) | A 批注后改原文 | 进「已归档」,主列表移除 |

> 后端侧的协同保证已由 `scripts/accept_v05.py` 覆盖：第 11 项(SSE 实时推送)+ 第 6/8 项(非作者 403 / 作者 200)。U1–U6 是把这些保证在**扩展 UI**上再确认一遍,属人工/E2E 范畴。

## 四、滑动续期（SQL，可选，单元测试已覆盖逻辑）

```bash
ssh aliyun 'sqlite3 /root/htmlGenius/annotations.db "SELECT substr(token,1,20)||\"…\", expires_at FROM sessions ORDER BY created_at DESC LIMIT 3"'
# 把某 token 的 expires_at 改到 30s 后
ssh aliyun "sqlite3 /root/htmlGenius/annotations.db \"UPDATE sessions SET expires_at=datetime('now','+30 seconds') WHERE token='<TOK>'\""
curl -s https://www.deuce.monster/hg/auth/me -H "Authorization: Bearer <TOK>"   # 触发 touch_session 续期
ssh aliyun 'sqlite3 /root/htmlGenius/annotations.db "SELECT expires_at FROM sessions WHERE token=\"<TOK>\""'
# 预期:expires_at 从「30s 后」跳到「~7 天后」
```

## 出问题自查

- **后端日志**：`ssh aliyun 'journalctl -u htmlgenius -f'`（callback 失败会 print 飞书原始返回）
- **服务状态**：`ssh aliyun 'systemctl status htmlgenius --no-pager | head -10'`
- **重启后端**：`ssh aliyun 'systemctl restart htmlgenius'`
- **某请求返回体**：侧边栏右键→检查→Network，看 `/hg/api/...` 状态码+响应
