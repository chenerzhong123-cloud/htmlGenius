# v0.6 元素级编辑（高级模式）设计

> 日期：2026-07-15 · 分支：`v0.6-visual-edit`
> 目标：在「文字编辑」之外，新增「元素编辑」模式——像 DevTools 那样悬停看框、点选元素、删除/复制/拖拽重排，让 AI 生成的静态 HTML 可视化编辑更灵活。

## 1. 范围

**v0.6 纳入（本设计）**
- 模式切换：文字模式 ↔ 高级模式（元素模式），互斥。
- 悬停 inspect：鼠标移过元素画框 + tooltip（标签/类名/尺寸）。
- 点选元素：点一下选中（非文字选区），持续高亮 + 侧边栏属性面板。
- 基础操作：**删除元素、复制元素**。
- **拖拽重排**：选中元素后在同级中拖动换序。
- 全部操作接入已有的撤销/重做（线性历史）。

**v0.6 不做（后续版本）**
- resize / 跨父容器移动（v0.7）。
- padding/margin/背景等样式编辑面板（v0.7，可与现有色环联动）。
- 语义识别（卡片/按钮/导航自动分组，v0.8+，可能结合 AI）。
- 框架页（React/Vue）兼容——**仅支持静态 HTML**（直接改 DOM 会和虚拟 DOM 打架，不在范围内）。

## 2. 模式架构

文字模式（现有）与元素模式（新）**互斥**，原因：
- 文字模式：`body.contentEditable = true`，点哪打字，靠 `execCommand`。
- 元素模式：要点选/拖拽元素，需拦截点击 + 画框，`contentEditable` 必须关掉，否则点击会进入文字编辑态。

**切换**：侧边栏右下角按钮「切换高级模式」，仅在 `_editing=true`（已开始编辑）时显示。点一下 → `toggle-element-mode` 消息 → content-script 切换：
- 开：`contentEditable=false` + 挂 inspect/click/pointer 监听 + 通知侧栏切换面板。
- 关：`contentEditable=true` + 卸监听 + 清 overlay + 恢复文字工具栏。

退出编辑（收起侧边栏 / 点退出编辑）时，元素模式一并关闭。

## 3. 核心交互

### 3.1 悬停 inspect
- `mousemove`（RAF 节流）→ `document.elementFromPoint(x,y)` 得目标元素。
- 跳过扩展自身注入元素（`#hg-toolbar`、`.hg-hl`、`.hg-inspect`、`.hg-select`）及 `body/html`。
- 复用一个 `.hg-inspect` div（`pointer-events:none`，否则挡住 elementFromPoint），定位到目标 `getBoundingClientRect()`。
- tooltip：`div.card · 320×180`（标签+首个类名 + 宽×高）。

### 3.2 点选元素
- `click`（元素模式）：`preventDefault` + `elementFromPoint` → `_selectedEl`。
- 画 `.hg-select`（持久，区别于 inspect 的颜色）。
- 发 `element-selected` 给侧栏：`{ tag, id, classes, w, h, siblingIndex, siblingCount, textPreview }`。
- `Esc` 或点空白 → 取消选择（选父元素：见 3.5）。

### 3.3 删除 / 复制
- 侧栏按钮 → 消息 `element-delete` / `element-duplicate`。
- content-script：先对当前正文快照，再 `_selectedEl.remove()` 或 `cloneNode(true)` 插到后面，然后**立即 `pushUndo()`** 把新状态压进历史。
- 复制后选中新副本。

### 3.4 拖拽重排（仅同级）
- 在选中元素上 `pointerdown` 记起点；移动 > 5px 判定为拖拽（否则视为点击选择）。
- 拖拽中：原元素半透明 + 一条 `.hg-drop-indicator` 指示线，落在「指针 Y 越过的同级中点」对应的缝隙。
- `pointerup`：`parent.insertBefore(_selectedEl, targetSibling)` 换序 → `pushUndo()`。
- **约束：只在同一父节点的子级间换序**（跨父容器移动留 v0.7，避免误操作）。

### 3.5 选父元素
- 属性面板「↑ 父元素」按钮 / `Esc` → `_selectedEl = _selectedEl.parentElement`（到 body 为止）。便于在嵌套结构里向上定位。

## 4. 数据/事件流

content-script → side panel：
- `element-mode-changed { on }`：模式翻转，侧栏切换面板。
- `element-selected { tag, id, classes, w, h, siblingIndex, siblingCount, textPreview }`。

side panel → content-script：
- `toggle-element-mode`
- `element-delete` / `element-duplicate` / `element-select-parent`

撤销：所有元素变更（删/复制/重排）后**同步 `pushUndo()`**，复用 v0.5.2 的线性历史，撤销/重做/还原初始 自动覆盖元素操作。

## 5. UI

- **「切换高级模式」按钮**：侧边栏右下角，`position:absolute/fixed`，仅 `_editing` 时显示。高级模式中变「退出高级模式」并高亮。
- **元素属性面板**（高级模式时替换 `#edit-tools` 显示）：
  - 头部：`div.card`（标签+类名）+ 尺寸 `320×180` + 同级位 `3/8`。
  - 正文预览：选中元素 `textContent` 前 ~60 字。
  - 操作行：`删除` `复制` `↑ 父元素`。
  - 提示行：「拖动元素可在同级中重排序」。
- inspect/选区/拖拽指示线用 Nebula 主题色（`--hg-brand` 边框 + 半透明底），跟现有 overlay 风格一致。

## 6. 边界与约束
- 仅静态 HTML；框架页提示「高级模式仅支持静态页」。
- overlay 全部 `pointer-events:none`（除拖拽时被选元素本身）。
- 跳过扩展注入元素与 `body/html`。
- 拖拽仅同级；跨容器移动不在 v0.6。
- 文字模式 ↔ 元素模式切换时，各自 overlay/工具栏显隐互斥。

## 7. 实施计划（里程碑）

| 里程碑 | 内容 | 产出 |
|---|---|---|
| **M1 地基** | `_elementMode` 状态 + `setElementMode` + `toggle-element-mode` 消息 + 右下角按钮 + contentEditable 挂/卸 + 模式互斥 | 能切模式，文字/元素不冲突 |
| **M2 inspect + 选区** | `.hg-inspect` 悬停画框+tooltip、`.hg-select` 点选、`element-selected` 消息 | 悬停看框、点选元素 |
| **M3 属性面板 + 删/复制** | 侧栏元素面板（信息+操作）、`element-delete/duplicate`、`pushUndo` 接入 | 选中后能删/复制，可撤销 |
| **M4 拖拽重排** | pointer 拖拽判定、同级 drop 目标计算、指示线、`insertBefore` 换序、`pushUndo` | 同级拖拽换序，可撤销 |
| **M5 收尾** | 选父元素(Esc)、空点取消、模式退出清理、i18n(中英日)、静态页检测提示、`node --check` + 实测 | 完成 v0.6 |

每个里程碑独立可测、独立提交。
