# htmlGenius v0.2 · HTML 实时编辑 — 设计文档

> 状态:Draft · 日期:2026-06-27 · 分支:`v0.2-html-editing` · 已整合架构 review 的 6 项必改

---

## 1. 背景与动机

v0.1(Beta)是「iframe 静态展示 HTML + 非侵入 overlay 批注 + 回灌 prompt」——人不能直接改 HTML,只能批注后让 AI 改。v0.2 要补「**人直接在渲染的 HTML 上实时编辑**」:演示/编辑时即时修错、调格式。编辑与批注并存,各司其职。

## 2. 目标与非目标

### 目标
- **G1 实时编辑**:整文档 contenteditable 常驻;光标改文字、浮工具栏调样式;改完即时渲染预览。
- **G2 破坏性 + 版本化**:编辑产出新版本(自动防抖存),保留滚动窗口历史,可回退。
- **G3 与批注并存**:编辑改 DOM、批注不改 DOM(overlay);编辑后批注自动 re-anchor;两者可叠加。
- **G4 结构安全**:contenteditable 不破坏 AI 复杂 HTML(清理 + 粘贴 sanitize + 限定容器)。

### 非目标(YAGNI)
- **不做协同/多人**(阶段 B)。
- **不做结构编辑**(增删/移动元素;只文本 + 样式)。
- **不引入富文本框架**(Tiptap/ProseMirror;用原生 contenteditable)。
- **不做 CLI 自动回灌**(仍是人复制 prompt)。

## 3. 核心决策(澄清结论)

| 维度 | 结论 |
|---|---|
| 编辑 vs 批注 | 并存:编辑**破坏性**(新版本、即时);批注**非侵入**(overlay、延迟回灌) |
| 编辑范围 | 文本(光标增删)+ 样式(加粗/颜色/字号/对齐/字体/背景)。**不改变块级布局结构**(p/h1/li/table 层级) |
| 版本 | 自动防抖(1.5s)+ 定时(5s 兜底)+ 滚动窗口(最近 20 版,超删旧)+ 存 db |
| 撤销 | **内存 undo 栈(前端,最近 50 步)≠ 版本表**;Ctrl+Shift+Z 回上一步;版本历史面板=显式回退 |
| 交互 | 整文档 contenteditable 常驻;**单一浮工具栏**(选中→Comment + 样式) |
| 批注触发 | 选中文字 → 浮工具栏 → 点「Comment」(批注该选区) |

## 4. 架构

v0.2 在 v0.1 viewer 上加「编辑运行时」。iframe 加载 HTML 后 `contentEditable=true` 常驻(限定在 `body` 内的内容容器,排除 head/script/style/注入元素)。两图层共用同一 iframe DOM:

- **编辑层(contenteditable)**:改 DOM 文本/样式 → 破坏性 → 产新版本。
- **批注层(overlay)**:`.ann-hl` 矩形,不改 DOM;编辑后 debounce re-anchor。

关键协调:editor.js 改 DOM 后 `emit('dom-changed')`;annotate.js 监听 → debounce 300ms `loadAnnotations()`(重 anchor + 判 stale)。

## 6. 关键技术决策(整合 review 必改项)

### 6.1 撤销:内存栈 ≠ 版本表(review 3.1/3.2/3.3)
- **内存 undo 栈**(前端,最近 50 步轻量 snapshot):Ctrl+Shift+Z 回上一步(**避开浏览器原生 Ctrl+Z** 的字符级 undo 死结)。
- **版本表**(db,滚动窗口 20):防抖/定时落库,持久化。
- **版本历史面板**:显式「时间旅行」(load version k → 还原),不绑快捷键。
- 防抖期间用户撤销:先 flush 当前 dirty 进 undo 栈再回退(version.js 暴露 `flushPending()`)。

### 6.2 序列化:完整文档 + 剥离注入(review 5.1/5.2/5.4)
- 存 `documentElement.outerHTML`(含 `<head><style>`,样式不丢)。
- 序列化前:clone 文档 → 移除所有 `data-htmlgenius-injected` 元素(overlay/浮层/注入 style)→ `blur()` + `removeAllRanges()`(去 caret 临时 `<br>`)。
- **schema 迁移**:`init_db` 检测 `versions.html_content` 列,缺则 `ALTER TABLE ADD COLUMN`;所有 INSERT 改**显式列名**。

### 6.3 contenteditable 结构安全(review 4.1/4.2/4.3)
- **粘贴**:`paste` 事件 `preventDefault` → `insertPlainText`(只插纯文本,剥离外部 style/class/script)。
- **input 清理白名单**:允许的标签 + style 属性白名单;不在白名单的节点 `unwrap`(保留 text)。`<pre><code>`、`<table>`、`<a>` 单独保真分支。
- **样式应用跨标签**:选中跨多元素时,`surroundContents` 拆分 + 边界 text node 处理(不是简单"改 selection 元素 style")。
- **限定可编辑**:`body` 内主内容容器 editable;`<head>/<script>/<style>` 及注入元素 `contenteditable=false` + `tabindex=-1`。
- 明确:**样式编辑会改结构**(包 span / 改 text-align);"不动结构"修正为"不动块级布局结构"。

### 6.4 编辑后 re-anchor + 软归档(review 1.1/1.2/1.3)
- editor 改 DOM → `emit('dom-changed')` → annotate debounce 300ms `loadAnnotations()`。
- `anchor()` 加**最低分门槛**:`bestScore < exact.length * 0.5` → 判 stale。
- **软归档**:anchor 失败不静默丢进归档区,而是主列表保留 + `stale` 徽章 + 「跳到上次定位处」入口。

### 6.5 单一浮工具栏(review 2.1)
- 合并 editor + annotate 对 `selectionchange` 的争夺:一个 `selection-toolbar.js`,同一选区一个 bar,内部分「Comment」(走 annotate)与「样式」(走 editor)。注入 iframe 内,与 v0.1 `#ann-float` 同源管理。

### 6.6 SQLite 并发与删版本(review 3.5/3.6)
- `init_db` 启用 **WAL**(`PRAGMA journal_mode=WAL`)。
- `add_version` 用**单事务**(读 current_version + 写 versions + 更新 documents)。
- 删旧版本前检查 `annotations.version`:有引用则把批注 `version` 更新到 `current_version`(简单)或保留该版本(软删)。

### 6.7 XSS:还原 sanitize(review 6.1)
- 还原版本时,HTML 经 `sanitize.js` 白名单清洗(去 `<script>`/onerror/onload 等)再写入 iframe。

## 7. 数据模型

`versions` 表加列:
```sql
ALTER TABLE versions ADD COLUMN html_content TEXT;  -- 完整文档 outerHTML(注入元素已剥离)
```
`VersionCreate` 模型加 `html_content: str`。其余沿用 v0.1(documents/annotations 不变)。

## 8. API

- `POST /api/documents/{id}/versions`(body: `{html_content, source}`)→ 新版本号(单事务 + 滚动窗口删旧)。
- `GET /api/documents/{id}/versions` → 版本列表(号、时间、source)。
- `GET /api/documents/{id}/versions/{v}` → 该版本 html_content。
- `DELETE /api/documents/{id}/versions/{v}` → 删(处理批注引用)。

## 9. 数据流

```
编辑(contenteditable 改 DOM / 浮工具栏改样式)
  → emit dom-changed → annotate debounce re-anchor(软归档判 stale)
  → version.js 防抖 1.5s / 定时 5s
  → 序列化(clone + 剥离 injected + blur)
  → POST 新版本(单事务 + WAL)
  → 滚动窗口(>20 删旧,处理批注引用)
撤销:Ctrl+Shift+Z → 内存 undo 栈 pop → 还原 snapshot DOM(flushPending 先入库 dirty)
版本回退:历史面板点 version k → GET html_content → sanitize → 写 iframe → 清 undo 栈
```

## 10. 错误处理

- **崩溃丢失**:定时 5s 兜底 + `visibilitychange`/`beforeunload` 强制 flush + `localStorage` 每 1s 镜像最近 dirty(启动恢复)。
- **并发**:乐观锁(`documents.updated_at`),冲突提示。
- **大文档**:html_content > 100KB 走 gzip 或文件存储(`versions/{doc}/{v}.html`,path 存库)。
- **序列化竞态**:序列化期间锁存( inflight flag)。

## 11. 测试

- **清理**:粘贴外部内容 → 白名单生效;Enter 在 p/li/td → 不破坏;跨标签选区样式。
- **序列化**:存版本含 head style;注入元素剥离;schema 迁移。
- **版本**:防抖存 + 滚动窗口删旧 + 删版本更新批注 version。
- **撤销**:Ctrl+Shift+Z 内存栈;防抖期间撤销 flushPending;版本历史回退。
- **re-anchor**:编辑后批注重 anchor;最低分门槛 stale;软归档徽章。
- **浮工具栏**:单一 bar(Comment + 样式);不与 v0.1 争夺。
- **sanitize**:还原去 script/onerror。
- **WAL/事务**:并发写不锁死。

## 12. 已知边界

- 单用户、本地(无协同;阶段 B)。
- contenteditable 清理对极端复杂 AI HTML 可能有未覆盖边角,靠白名单 + 保真分支渐进收敛。
- 撤销栈内存级(刷新失效);持久历史靠版本表。
- 还原 sanitize 可能改写部分内联事件(可接受,安全优先)。
