# htmlGenius v0.3 · Chrome 插件 — 设计文档

> 状态:Draft · 日期:2026-06-30 · 基于 v0.2.0-self 基线 + 架构 review 修订

## 1. 背景与动机

v0.2 是 web app（FastAPI + SQLite + iframe viewer），只能编辑 samples/ 下的 HTML。用户希望作为 Chrome 插件，编辑/批注任意网页——不限目录、不限文件。

经架构 review（7 维度质疑），核心洞察：**编辑"别人网页"语义不成立**（刷新丢、React 覆盖、版本还原伪功能）。方案改为**双模式**：本地全功能编辑、远程只读批注。

## 2. 目标与非目标

### 目标
- **G1 Chrome MV3 插件**：side panel 侧边栏 + content script 注入当前页。
- **G2 双模式**：本地（file/localhost/data/blob）默认可编辑；远程默认只读批注，手动「编辑」入口 + 提示。
- **G3 批注 + 回灌**：任意网页选区批注（overlay）+ 回灌 prompt 复制剪贴板（核心，两模式都有）。
- **G4 复用 v0.2 核心**：text-quote anchoring、DOMPurify sanitize、selection-toolbar 批注入口。

### 非目标(YAGNI)
- **不做飞书式 Y 锚定**（side panel 独立窗口做不到卡片随网页滚动）→ 静态排序 + 点击跳转。
- **不做远程版本还原**（无法写回第三方网页）→ 远程模式无版本管理。
- **不申请 `<all_urls>` 权限** → 用 `activeTab`（点图标才授权）。
- **不做 background 中枢** → content script ↔ side panel 直连 `chrome.runtime.connect`。

## 3. 双模式判断与交互

### 判断逻辑
```
location.protocol === 'file:'                    → 本地
location.hostname === 'localhost' / '127.0.0.1'  → 本地
location.protocol === 'data:' / 'blob:'          → 本地
其他 (http/https 远程域名)                         → 远程
```

### 交互
| 模式 | 默认状态 | 编辑入口 | 版本 | 批注 | 回灌 |
|---|---|---|---|---|---|
| **本地** | contenteditable 自动开 | 无需（默认编辑） | ✅ IndexedDB 存版本 + 还原 | ✅ | ✅ |
| **远程** | 只读（不 contenteditable） | 手动点「编辑」→ 弹提示后开启 | ❌（仅当前会话） | ✅ | ✅ |

**远程编辑提示**：「⚠ 编辑仅本地临时修改，刷新或关闭页面后丢失，无法保存回原网页。」用户确认后才进编辑。

## 4. 架构（Chrome MV3）

```
┌─ side panel(独立窗口)          批注卡片(静态排序) + 版本历史(本地模式) + 回灌按钮 + 模式指示
├─ content script(注入当前页)    双模式判断 → 编辑(contenteditable,仅本地) + overlay 批注 + toolbar
│                                 text-quote anchoring + DOMPurify sanitize + selection-toolbar
├─ background service worker      安装事件 / activeTab 权限 / 跨 tab 同步(不做消息中枢)
└─ IndexedDB                     annotations store(批注,所有模式) + versions store(版本,仅本地模式)
```

## 5. 关键设计决策(review 修订)

### 5.1 overlay 不进 Shadow DOM
overlay 高亮（`.ann-hl`）必须和被标注文本**共享坐标系**（position:fixed + getBoundingClientRect）。只有 UI（浮工具栏、批注卡片）进 Shadow DOM 防样式污染。

### 5.2 卡片静态排序（放弃飞书 Y 锚定）
side panel 是独立窗口，做不到卡片随网页滚动 Y 同步。改为：
- 卡片按文档顺序（anchor 时的 DOM 位置）静态排列。
- 点卡片 → `chrome.runtime.sendMessage` → content script `scrollIntoView` + flash overlay。

### 5.3 overlay 定位改 position:fixed
v0.2 overlay 用 `position:absolute + scrollY`（iframe 单一滚动）。插件当前页有多个滚动容器 / fixed / sticky → 改 `position:fixed` + 实时 `getBoundingClientRect`（视口坐标，不加 scrollY）。rAF 更新位置。

### 5.4 document_id = origin + pathname（剥 query）
```javascript
const docId = location.origin + location.pathname;  // 不含 query/hash(?)
```
SPA hash 路由保留 hash（`/#/article`）。UI 明示「动态内容，批注可能失效」。批注 stale 进归档（v0.1 已有 archive）。

### 5.5 通信:chrome.runtime.connect port(不做中枢)
content script ↔ side panel 用 port-based 长连接（`chrome.runtime.connect`）。background 不做消息转发（MV3 service worker 30s 回收，转发会丢消息）。

### 5.6 权限:activeTab
manifest 只申请 `activeTab`（点图标授权当前 tab），不申请 `<all_urls>`。牺牲自动注入，换商店审核 + 用户信任。

### 5.7 IndexedDB 数据模型
**annotations store**(所有模式):
```javascript
{ id, document_id(url), selector{exact,prefix,suffix}, quote, body{comment,action,instruction}, created_at, status }
```
索引:`document_id`。

**versions store**(仅本地模式):
```javascript
{ id(auto-increment), document_id, html_content, created_at, source }
```
索引:`document_id`。滚动窗口 20 版。

## 6. 数据流

### 批注创建(两模式)
选区 → content script `describe(range, document.body)` → selector → IndexedDB annotations.put → postMessage side panel → 卡片刷新。

### 批注渲染(overlay)
加载 → IndexedDB annotations.get(document_id) → 对每条 `anchor(selector, document.body)` → position:fixed 矩形覆盖 → rAF 更新位置(滚动/resize)。

### 编辑(仅本地模式)
contenteditable → input → IndexedDB versions.put(防抖 1.5s, 滚动窗口 20) → undo 内存栈(Ctrl/Cmd+Z)。

### 回灌
side panel「回灌」→ content script 取批注 → buildPrompt(三段联合) → 复制剪贴板。

### 远程编辑入口
side panel / content script「编辑」按钮 → 判断远程 → 弹提示 → 确认 → contenteditable=true(仅当前会话,不存版本)。

## 8. 已知边界
- **SPA 页面批注失效**:URL 不变但 DOM 变 → TextQuote 找不到 → stale 归档。UI 明示。
- **远程编辑临时性**:刷新丢失,无法保存回原网页(提示已说明)。
- **嵌套 iframe**:content script 不注入子 iframe,跨 iframe 选区拿不到。
- **Shadow DOM 目标页**:若目标网页本身用 Shadow DOM(web components),getSelection 拿不到 shadow 内选区。
- **大型网页性能**:buildIndex 全量遍历文本节点,大网页(数万节点)可能卡。后续加缓存 + 失效。
- **Chrome Web Store 审核**:activeTab + sidePanel 权限,需说明用途。

## 9. 与 v0.2 的关系
- v0.2 web app(FastAPI + SQLite + viewer)**保留**(编辑自己 HTML 的本地工具)。
- v0.3 Chrome 插件是**独立产品**(批注/编辑任意网页)。
- 共享核心:text-quote.js + sanitize.js(两份 copy 或提取为共用模块,后续决定)。
- v0.2.0-self tag 是安全基线。
